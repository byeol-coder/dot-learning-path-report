01

조사 결과

서비스 베이스와 교육과정 데이터를 분리해 읽고, 기존 기능을 유지하는 조건에서 접근성 학습 경로를 얹었습니다.

서비스 베이스

byeol-coder/dot_lens, HEAD 1f98d20 기준입니다. Next.js 15 정적 export 앱이며 프로덕션 basePath=/dot_lens를 사용합니다. 핵심 재사용 모듈은 lib/lessonPlayer.ts, lib/tactileScene.ts, lib/dotpadDevice.ts, lib/brailleEngine.ts, 접근성 프로바이더, useLang() i18n입니다. 기존 기능과 라우트는 제거하거나 약화하지 않았습니다.

교육과정 데이터

DECK6/korean-elementary-learning-map, 커밋 3ef0563... 기준입니다. MIT License, NOTICE.md 원저작 표시 요구가 있으며 교육부·NCIC 공식 문서가 아닙니다. 주제 1,956개, 클러스터 153개, 선수 관계 1,894개, 성취기준 620개를 읽기 전용 외부 데이터로만 취급했습니다.

02

제품 설계

짧은 목표, 즉각 피드백, 이어하기, 복습 추천, 부드러운 일일 목표는 차용하고 징벌적 장치는 의도적으로 배제했습니다.

  • 잠금 대신 추천
  • 힌트 사용도 완료 인정
  • 하트 없음
  • 공개 랭킹 없음
  • 위협적 연속 학습 문구 없음
  • 화면·점자·음성·키보드 동시 지원
  • AI/초안 콘텐츠 draft 표시

없는 데이터는 정직하게 라벨링했습니다. “촉각 학습 가능”, “텍스트·음성 학습 가능”, “준비 중”, “검수 필요” 상태를 구분하고, 가짜 시작 버튼은 두지 않았습니다.

03

구현 결과

데이터 어댑터 계층

lib/curriculum/ 아래에서 buildCurriculum.mjs가 고정 SHA에서만 스냅샷을 생성합니다. 다른 커밋이면 중단하고, sha256 해시와 출처 문구를 manifest에 기록하며, public/curriculum/에 커밋해 오프라인 빌드를 보장합니다.

촉각 레슨 8종

수 크기 비교, 수직선 덧셈, 여러 가지 삼각형, 여러 가지 사각형, 식물의 부분, 물의 세 가지 모습, 달의 모양 변화, 지도의 방위를 손으로 설계한 60x40 매트릭스와 이중언어 설명·힌트·퀴즈로 구성했습니다.

학습 화면

/learn/home, /learn, /learn/lesson/[id]를 추가했습니다. 키보드, 화면 버튼, 패닝키가 하나의 상태 머신을 구동하고 점자 라인, TTS, 기기 미러링, 시뮬레이션 경로를 제공합니다.

접근성 상태 표현

Esc 음성 중지, 이어하기 복원, 결과 화면의 “배운 내용 먼저·점수 나중” 구조, 색상 단독이 아닌 모양과 텍스트 기반 상태 표시를 반영했습니다.
